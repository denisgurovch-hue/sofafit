# Fix: модалка виджета прыгает при hover на карточку

## Причина

Сниппет создаёт модалку с `position: fixed` **внутри** `.widget-mount`, который сейчас лежит внутри `<article>` карточки товара. У карточки есть hover‑трансформации:

- `hover:-translate-y-1` на `<article>`
- `group-hover:scale-[0.7]` на `<img>`

По спецификации CSS, если у любого предка есть `transform`, то `position: fixed` у потомка ведёт себя как `position: absolute` относительно этого предка. В итоге открытая модалка "цепляется" за карточку и дёргается при hover.

Кроме того `overflow:hidden` на `.product-main-image` и общий `overflow-hidden` на `<article>` потенциально могут клипать всплывающие элементы сниппета.

## Решение

Разделить две вещи, которые сейчас слиты в `.widget-mount`:

1. **Кнопка "AI Try"** — должна оставаться в карточке (внутри `.product-body`).
2. **Модалка** — должна жить вне карточки, чтобы её `position:fixed` корректно позиционировался относительно viewport.

Сниппет монтирует и кнопку, и модалку в один и тот же `mountSelector`, поэтому проще всего:

- Оставить `.widget-mount` в карточке как место для **кнопки**.
- После инициализации виджета программно **переместить саму модалку** (`.fi-modal`) из карточки в `document.body`. Кнопка остаётся в карточке и продолжает работать (обработчики на ней навешены сниппетом и не зависят от положения модалки в DOM).

Это минимально‑инвазивно: не трогаем сниппет, не ломаем верстку карточки, чиним именно симптом.

## Изменения

### `src/lib/useFurnitureWidget.ts`

После `widget.init(...)` для каждой карточки найти созданную сниппетом модалку внутри `.widget-mount` и перенести её в `document.body`:

```text
for each cardId:
  init(...)
  const root = document.getElementById(id)
  const modal = root.querySelector('.fi-modal')
  if (modal && modal.parentElement !== document.body) {
    document.body.appendChild(modal)
  }
```

Это уберёт модалку из‑под трансформированных предков, и `position:fixed` снова станет относительно viewport.

При размонтировании компонента (cleanup) — удалить перенесённые модалки из `document.body`, чтобы при повторном монтировании не оставались дубли. Помечаем перенесённые модалки атрибутом `data-fi-card="<cardId>"` для надёжной очистки.

### `src/components/storefront/ProductCard.tsx`

Изменений по сути не требуется — `.widget-mount` остаётся, в нём будет жить только кнопка. Опционально можно убрать у `<article>` `overflow-hidden` (оставить overflow только на `.product-main-image` для эффекта зума картинки), чтобы исключить любые будущие клипы всплывающих элементов кнопки. Это безопасно: скруглённые углы карточки сохранятся за счёт `rounded-xl` на дочерних блоках/изображении.

## Технические детали

- Сниппет уже навешивает обработчики на кнопку и слушатель `keydown`/`message` на `document`/`window`, поэтому перенос модалки в `body` не ломает логику открытия/закрытия.
- Идентификация модалки конкретной карточки: после `init` мы знаем `#${cardId} .widget-mount .fi-modal` — берём её **до** переноса.
- Cleanup в `useEffect` удаляет все `[data-fi-card]` модалки, созданные этим хуком, чтобы StrictMode / повторные ре‑рендеры не плодили дубли.

## Файлы

- `src/lib/useFurnitureWidget.ts` — добавить перенос модалок в `body` + cleanup.
- (опционально) `src/components/storefront/ProductCard.tsx` — убрать `overflow-hidden` с `<article>`.
